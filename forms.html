Search Form

The RyzeUI search form has been constructed to include all UI components necessary for a rich user experience. It takes advantage of flexbox which you can read more about here. Flexbox works on all modern browsers, see which here.

Search Demo

HTML

<form role="form" class="search">
  <input type="search" required placeholder="Search RyzeUI">
  <button class="close-icon" type="reset"></button>
  <button class="btn btn-default">Go</button>
</form>

Note. For the reset button to work, the input required attribute is required.

Default Inputs

RyzeUI supports all text-based input fields. Included support for all HTML5 types: text, password, date, time, number, email, url, search, tel, and color.

Demo

HTML

<input type="text" placeholder="Standard text input">

Note. RyzeUI does not require default inputs to have a class.

Textareas

Textareas can be rezised vertically when the resize handle is dragged. Horizontal scaling has been disabled. The textareas support multiple lines using the rows="" attribute.

Demo

HTML

<textarea rows="4" placeholder="Standard text input"></textarea>

Selects

RyzeUI has opted to used default browser styling with absolute minimal changes made for best cross-browser compatibility.

Demo

HTML

<select>
  <option selected>Option 1</option>
  <option>Option 2</option>
  <option>Option 3</option>
  <option>Option 4</option>
  <option>Option 5</option>
</select>

For multiple selects, we have chosen to adopt similar styling to our other default inputs.

Demo

<select multiple>
  <option selected>Option 1</option>
  <option selected>Option 2</option>
  <option>Option 3</option>
  <option>Option 4</option>
  <option>Option 5</option>
</select>

For styled selects, wrap your standard select with a div ensuring it has the class select-ryze.

Demo

HTML
<div class="select-ryze">
  <select>
    <option selected>Option 1</option>
    <option>Option 2</option>
    <option>Option 3</option>
    <option>Option 4</option>
    <option>Option 5</option>
  </select>
</div>

Disabled State

Some forms require you to submit data that cannot be changed. In this circumstance you may add the disabled attribute to any input, select or select.

Demo

HTML

<input type="text" placeholder="Disabled" disabled>

Readonly Inputs

Some forms may ask the user to fill out something previously submitted that cannot be changed. In this circumstance you may add the readonly attribute to any input.

Demo

HTML

<input type="text" value="Ryze UI" readonly>
